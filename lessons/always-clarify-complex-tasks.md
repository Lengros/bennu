# Always Clarify Complex Tasks
date: 2026-03-19
scope: process
rule: Clarify ideal outcome, distortion, and need validation even with a pre-formed user solution; skip only on explicit 'just do it.'

## Why
A pre-formed user solution (tool choice, epic breakdown, architecture) was taken at face value without clarification. The implementation matched the stated solution but missed the actual need. "Let's discuss step by step" IS the clarification, not a skip signal.

## How to apply
For any task with meaningful complexity: name the ideal outcome, check whether the proposed solution might be distorted from the real need, validate that the need is real before decomposing. The clarification surfaces assumptions that are invisible inside a pre-formed plan.
