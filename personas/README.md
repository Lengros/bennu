# Personas — seeds, not a menu

These cards are **starting points**, not a fixed roster to pick from.

**Generate-first doctrine:** before casting, reason about the competencies THIS task
actually needs. Then check these cards for a fit. If none fits cleanly — **synthesize a
better persona on the spot** (blend two cards, sharpen the expertise, add task-specific
standards). Do not contort the task to match an existing card.

**Save reusable syntheses back.** If a synthesized persona is general enough to recur,
write it as a new card here — one persona per file, same structure as the existing cards
(`# id`, `role:`, `expertise:`, `standards:`, `anti-patterns:`, `output:`, `use-when:`),
≤15 lines. One-off syntheses stay in the brief; they don't need a file.

**Card structure:** `# id`, `role:`, `expertise:`, `standards:` (2-4 bullets, what
"good" means), `anti-patterns:` (2-3 bullets, failure modes to avoid), `output:`
(artifact shape + evidence tags: `[Observed:]`/`[Inferred:]`/`[Hypothesized]`/`[ASSUMPTION]`),
`use-when:`. A missing persona is never an excuse to skip delegation — synthesize one.
