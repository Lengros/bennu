# "Convenient" Means Lifetime User Friction, Not Build Cost
date: 2026-06-08
scope: product
rule: Rank product options by total lifetime user friction; build cost is a separate axis; never let cheapest-to-build masquerade as convenient.

## Why
"Cheapest to build" was presented as "most convenient for users." The option that required the least build cost required recurring manual user effort. Tools people return to require one action ever and then maintain themselves. Recurring friction dominates lifetime experience; one-time self-renewing credentials beat any manual treadmill.

## How to apply
On any product-design question: (1) state the experience bar first (what does "convenient" actually mean for this user over their lifetime with the product?); (2) rank options by total user-experienced friction — onboarding cost plus recurring maintenance; (3) present build cost as a separate, explicitly-labeled axis. Never conflate the two in the summary or recommendation.
