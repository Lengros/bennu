# No Task Is Undelegable
date: 2026-04-06
scope: delegation
rule: No catalog match means construct custom competencies via executor synthesis; 'no matching profile' is never an excuse to embody domain work.

## Why
Curator attempted infra/tooling work inline on grounds that no catalog profile covered it. The REASON step of executor synthesis is exactly where you derive a custom profile — skipping it is a rationalization, not a constraint.

## How to apply
REASON (what competencies does this task require?) → if no catalog match, list the required competencies explicitly → use them directly in the subagent brief as persona framing. Infra, auth, scripts, CI/CD work are delegable domain work. "Faster to do it myself" is always wrong at Curator level.
